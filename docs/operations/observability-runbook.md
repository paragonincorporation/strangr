# Observability and cost-control runbook

Before enabling public traffic, H16 must connect the provider-neutral counters to an organization-owned error, uptime, analytics, paging, and budget system. Do not send message text, profile fields, tokens, exact birth dates, media, IP addresses, Stripe payloads, or moderator evidence.

Alert/runbook mapping: readiness or WebSocket failures → incident commander; queue wait or match-success regression → engineering and safety; TURN/database/Redis/email/storage/billing spikes → engineering plus finance owner; webhook delay or worker lag → billing/on-call. Every alert needs an owner, severity, acknowledgement target, rollback/kill-switch, and safe evidence link in the private register.

Set global and country concurrency ceilings only after Task 20 load evidence and staffed moderation approval. When a ceiling is reached, show the beta-full state; do not silently queue indefinitely or evade country controls.
